Managing Sales Pipelines and Opportunities

An Opportunity in Siebel represents a potential revenue-generating event. The sales pipeline is the process of guiding an opportunity through various Sales Stages, from initial qualification to closing the deal.

Opportunity Attributes:

  • Sales Stage: Defines the current phase of the deal (e.g., 01-Prospecting, 03-Closing, 05-Won). This often drives the expected timeline and resources required.
  • Probability: A key metric that is often driven by the Sales Stage. This is used for forecasting.
  • Revenue: Tracks the expected or committed value of the deal.
  • Sales Team (MVG): An Multi-Value Group (MVG) that links the Opportunity to multiple **Positions** (people) involved in the sale.

Forecasting and quota management

The Siebel Forecasting module allows management to collect, consolidate, and analyze projected sales data against assigned targets.

Key Concepts:

  • Hierarchical Roll-up: Forecasts submitted by individual Sales Reps are automatically **aggregated** up the **Position Hierarchy** to provide sales managers and executives a consolidated view of projected revenue for their entire reporting structure.
  • Forecasting Periods: The system allows for setting targets (quotas) and collecting forecasts based on predefined time periods (e.g., quarterly or annual).
  • Data Model: Forecasting data uses dedicated tables and BCs (e.g., `S_FORECAST`), allowing for separation from the core Opportunity records.

Creating and managing marketing campaigns

Siebel's campaign management is geared toward multi-stage, multi-channel execution and detailed response tracking for accurate ROI.

Campaign Lifecycle:

  • Design: Campaigns are designed to target specific **Segments** (audiences) and execute a series of steps (e.g., sending an initial email, followed by a follow-up call if no response).
  • Execution: Campaigns interact with various channels (Web, email, print) to deliver personalized offers.
  • Response Tracking: The system tracks customer response (e.g., a purchase, a form fill) and links it directly back to the originating Campaign ID and the Segment ID.
  • Lead Creation: A successful campaign response can automatically create a **Lead** record, passing all relevant data to the sales process.

Lead generation and nurturing

The Lead Management process focuses on converting raw inquiries into qualified sales opportunities by efficiently routing and tracking prospects.

The Lead Management Process:

  • Capture and Qualification: Leads are captured from various sources. **Qualification Rules** (often declarative expressions or Workflows) are applied to determine if the lead is sales-ready.
  • Routing (Assignment Manager): Qualified leads are automatically routed to the correct Sales Rep or Sales Team using **Assignment Manager**. The routing rules consider factors like territory, product interest, and Rep workload.
  • Nurturing: Leads that are **not** sales-ready are routed to the marketing department, where they are enrolled in **Nurturing Campaigns** until they become qualified.
  • Conversion: A qualified Lead is converted into an **Opportunity** and **Contact** record, transferring all necessary data to the sales process.

Managing customer support and service operations

Siebel Service Management extends the Call Center functionality to include full **Field Service** capabilities, critical for organizations that service physical assets at customer locations.

Field Service Lifecycle:

  • Asset Tracking: Service is managed against customer **Assets** (installed products). The Asset record tracks the product's serial number, configuration, and service history.
  • Dispatch and Scheduling: Service Requests are converted into **Activities** (work orders). The system automates the **dispatch and scheduling** of mobile field technicians based on technician **Skills**, **Location**, and current **Workload**.
  • Inventory: Tracks parts required for repairs, managing inventory levels in regional warehouses and on individual service vehicles.

Configuring service level agreements (SLAs)

Service Level Agreements (SLAs) define contracted performance metrics (like response or resolution time) for specific customers or service types. Siebel monitors these commitments in real-time.

SLA Management:

  • SLA Definition: SLAs are defined by linking **Service Contracts** (what the customer bought) to **Entitlements** (what they are owed). The SLA specifies time-to-respond and time-to-resolve targets.
  • Automated Monitoring: Siebel automatically starts timing an SR as soon as it's logged.
  • Breach Prevention: **Workflow Policies** are typically configured to monitor the remaining time on an SLA. If the time limit is approaching, the system triggers an **Automated Escalation** (e.g., sending an alert email to the manager).

Integrating Siebel CRM with ERP and other applications

Siebel **EAI (Enterprise Application Integration)** provides a comprehensive platform for connecting Siebel to external systems (ERP, legacy databases, cloud services).

Key Integration Technologies:

  • Siebel Business Services: These are reusable, callable functions that perform specific tasks. Services like **EAI Siebel Adapter** (to read/write Siebel BC data) and **EAI HTTP Transport** (to make web service calls) are used to interact with outside systems.
  • Integration Objects: A metadata definition in Siebel Tools that represents a Siebel Business Object in a flat, hierarchical structure (XML) for easy data exchange. The Integration Object defines the common language for integration.
  • Web Services: Siebel can consume and expose **SOAP** and **REST** Web Services, allowing real-time, synchronous integration with other applications.

Ensuring seamless data flow between systems

For large-scale, asynchronous integrations, using **EIM** remains the standard. For real-time flows, careful modeling and **error handling** are paramount.

Real-time Integration Pipeline:

  • Data Mapping/Transformation: Data traveling between systems is transformed using declarative mapping or **Integration Workflow Processes**.
  • Business Service Invocation: A **Workflow Process** orchestrates the entire flow, calling the **EAI Siebel Adapter** to get data and then calling a **Transport Business Service** to send the data out.
  • Error Handling: The integration workflow must include robust error steps to log and notify administrators if the external system fails or returns an invalid payload.

Using Siebel CRM on mobile devices

Siebel provides specialized client types to ensure users can access the CRM when disconnected from the network or using mobile devices.

Siebel Client Types:

  • Siebel Web Client (Web Browser): The primary client. Accesses the Siebel Enterprise via a web server (synchronous).
  • Siebel Mobile Web Client: A local executable installed on a laptop. It accesses a Local Database (`sse_data.dbf`) and a Local SRF, allowing users to work **disconnected**. Data is periodically synchronized with the server.
  • Siebel Developer Web Client: Used by developers/administrators for direct access to the Siebel database/repository.

Mobile app configuration and usage

Mobile access often requires separate configuration of screens and applets to ensure an optimal touch-friendly experience on small form factors.

Mobile Configuration:

  • Streamlined Application: Administrators may define a separate **Application Object** (in Siebel Tools) designed specifically for mobile users, including only the essential screens and views.
  • Mobile Web Templates: Mobile views and applets use specialized **Web Templates** designed for touch interaction and responsive behavior.
  • Synchronization: Mobile users synchronize their data using the **Synchronization Manager** server component, pushing local changes to the server and downloading updates from other users.
