Creating and customizing reports

Reporting provides static, printable documentation of business data. Siebel leverages external tools for advanced report generation.

Report Types:

  • Standard Reports: Simple, pre-built reports often accessible from the application toolbar.
  • Oracle BI Publisher: For complex, pixel-perfect reports (e.g., invoices, legal forms), Siebel integrates with **BI Publisher**. Report design templates are created externally and uploaded to the Siebel server.
  • Customization: Customizing a report involves modifying the external template and ensuring the **data mapping** (the query that pulls the data from Siebel) is correctly defined.

Siebel Reporting Flow

Using dashboards and visualizations

Analytical capabilities often rely on integrating Siebel data with a sophisticated business intelligence platform like **Oracle Business Intelligence Enterprise Edition (OBIEE)**.

Analytical Tools:

  • Data Warehouse: For historical analysis, Siebel data is typically extracted and loaded into a separate **Data Warehouse**. OBIEE queries this warehouse, not the live transaction database.
  • Interactive Dashboards: OBIEE provides dynamic dashboards with **Key Performance Indicators (KPIs)**, charts, and visualizations that allow drill-down and filtering.
  • Embedding: These analytical dashboards can be embedded directly into the Siebel Web Client UI, providing users with seamless access to both transactional and analytical data.

Key performance indicators for CRM

Defining the right **Key Performance Indicators (KPIs)** is essential for measuring the effectiveness of business processes and driving continuous improvement.

Common CRM KPIs:

  • Sales KPIs: Include Lead Conversion Rate, Sales Cycle Length, Average Deal Size, and **Quota Attainment** (performance against targets).
  • Service KPIs: Include **First Call Resolution Rate** (percentage of issues solved on the first call), **Average Handle Time**, and Customer Satisfaction Score (CSAT).
  • Implementation: KPIs are implemented as calculated fields on business components or as metrics within an integrated BI platform.

Monitoring and analyzing performance metrics

Dashboards and reports are the primary tools for consistently monitoring KPIs and analyzing performance metrics.

Operational Focus:

  • Management Dashboards: Provide a high-level overview of key metrics for executives and managers.
  • Operational Reports: Provide detailed, tactical information for front-line employees to manage daily work.
  • Analysis: Analyzing trends over time allows the business to identify areas for improvement and make strategic adjustments.

Ensuring data security in CRM

Security is enforced through multiple layers, starting before the user even gains access to the application.

Security Layers:

  • Authentication (Who you are): Verifying the user's identity through login credentials. Siebel supports database login, LDAP, and **Single Sign-On (SSO)** via a security adapter.
  • Authorization (What you can see/do):
    • Functional (Responsibilities): Controls access to Views (functional areas).
    • Data (Positions/Organizations): Controls access to Records (data subset).
  • Communication Security: Using **HTTPS/TLS** protocols to encrypt data transmitted between the client (browser), Web Server (SWSE), and Siebel Server.

CRM Security Layers

Validation: Log in as two different users with different **Responsibilities** (functional access) and note which top-level tabs appear. Then, within the same tab, observe the different records shown (data access).

Compliance with industry standards and regulations

Siebel provides specialized features to help organizations adhere to strict data compliance and audit regulations (e.g., SOX, GDPR).

Audit and Compliance Tools:

  • Audit Trail: A critical feature that, when configured, automatically tracks every change made to a sensitive data field. It logs the **old value**, **new value**, **who** changed it, and **when** it was changed, providing a complete history for auditing.
  • Data Masking/Encryption: Siebel supports encryption techniques to protect highly sensitive data fields both when stored in the database (**at rest**) and while being transmitted (**in transit**).

Compliance and Audit Trail

Validation: Navigate to **Administration - Audit Trail** screen. Here, you define which Business Components and Fields should be monitored for changes, enabling the audit trail feature.

Implementing best practices for CRM operations

Success in CRM is achieved when technology, processes, and people align. Best practices focus on leveraging Siebel's declarative tools to automate and enforce business processes consistently.

Operational Focus:

  • Standardization: Document and define a **standardized process** for every major activity (e.g., a 5-step lead qualification process).
  • Automation & Enforcement: Implement this standardized process using **Workflow Processes** and **State Models** to guide users and ensure compliance every time the activity occurs.
  • User Training & Adoption: Continuous training, a clean UI (Customized Applets/Views), and fast system performance are critical to ensuring users actually use the system.

Common issues and solutions

Troubleshooting in a Siebel enterprise is systematic, often involving the analysis of logs generated by the server components.

Troubleshooting Toolkit:

  • Server Management: The **Administration - Server Management** screen provides the status of all Siebel Servers and running **Server Components** (like the Object Manager).
  • Log File Analysis: For component errors, administrators increase the **Log Level** for the component (from 0:Fatal to 5:Diagnostic) to gather detailed diagnostic information in the server log files.
  • Component Restarts: Changes to certain **Server Parameters** (like a database connection string) may only take effect **At Server Re-Start** or **At Component Re-Start**.

Server Management & Logging

Validation: Navigate to **Administration - Server Management > Components**. Select the **Workflow Monitor Agent** component and check its parameters to adjust the logging level for troubleshooting.
