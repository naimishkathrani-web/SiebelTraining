What is Customer Relationship Management?

CRM is the system that centralizes customer data and processes so teams can sell, service and market consistently.

Clear concept

It records contacts, interactions, contracts, orders and support cases. That single record avoids multiple disconnected spreadsheets and ad-hoc emails.

Practical example (telecom)

A service agent sees a subscriber's order, billing and past incidents on one screen and resolves the ticket faster.

Why CRM matters to the business

CRM increases customer retention, helps create targeted campaigns and improves productivity for agents and salespeople.

Business Impact — Simple reasoning

When customer history is accurate and accessible, decisions are faster and mistakes decrease — this improves customer satisfaction and revenue outcomes.

Architecture — Server View & 3-Layer Logical View

Understand both the physical server components and the logical UI/Business/Data layers.

Server Architecture (physical)

Components: Web Server + SWSE → Siebel Enterprise Server (Object Managers, Workflow Engine, EIM) → Oracle Database. BI Publisher (reporting server) is invoked by the Siebel server when formatted reports are requested.

Request flow: Browser → Web Server (SWSE) → Siebel Web Client → Object Manager → DB. Report generation flows via a Report Server call to BI Publisher for PDF/XLS rendering.

Image placeholder — Images/siebel_architecture.png
Source: crop from D51681 (search "Architecture") or Oracle Bookshelf

3-Layer Logical View

  • UI Layer: Applets, Views, Screens — user interaction surface.
  • Business Layer: Business Components, Business Objects, Workflows, Business Services — handles business rules and processing.
  • Data Layer: DB tables, EIM staging, audit tables — persistent storage.

Each layer is designed for separation of concerns: UI changes do not require database redesign and vice versa.

Understanding the Data Model (simple)

Business Components map UI to tables; Business Objects group related BCs for screens.

Concept

BCs define fields, joins and search specs. Changing a BC controls how data appears in applets without changing database code.

Navigating the Application

Responsibilities & Access Control

Best practice
