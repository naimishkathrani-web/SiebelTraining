Business Components (BC) — Concept

A Business Component is the logical representation of data that an Applet uses. BCs map to tables and define the fields visible in the UI and search behavior.

Detailed explanation

BC properties include Table, JoinMap, SearchSpec and Field definitions. You can add calculated fields, mapped columns, and set read-only or computed attributes via BC configuration.

Est. Time: 12 mins

Business Object (BO) — Grouping BCs

BOs group related BCs to form a screen or user flow. They determine master-child relationships and navigation within a screen.

Example

Account BO might include Contact BC, Opportunity BC and Service Request BC as related BCs to show linked records on the same screen.

Est. Time: 10 mins

Fields, Extension Tables & Safe Customizations

Use extension tables for custom fields (prefixed with X_) to avoid altering core tables. Map fields in BCs and add them to Applets.

Best practice

Create extension tables for any custom data. This keeps upgrades and patching safe and minimizes conflicts.

Est. Time: 12 mins

Views & Applets — Presenting BC data

An Applet displays BC fields in list or form mode. Views arrange one or more Applets; Screens present Views to users based on responsibility.

Actionable steps

To add a field to a view: add to BC → add field to Applet → save to workspace → deliver to Integration for QA.

Est. Time: 12 mins

SearchSpec & Query Behavior

SearchSpec determines which rows appear in lists. Understand index-friendly filters and avoid expensive server-side computations in search specs.

Why it matters

Efficient searches ensure responsive UI and lower DB load. Use indexed columns where possible and keep SearchSpec simple.

Est. Time: 10 mins

Calculated & Derived Fields

Use calculated fields for UI-level derived data (e.g., FullName) or use Business Services for heavier calculations.

Example

FullName = FirstName + ' ' + LastName can be a calculated field in BC to avoid duplicating logic in the UI.

Est. Time: 8 mins

Applets: List vs Form Behaviors

List applets support multi-row operations and bulk actions; form applets present detailed fields and granular actions.

Tip

Keep list applet columns lean (key columns) and reserve detailed or rarely-used fields for form applets.

Est. Time: 8 mins

UI Performance Considerations

Large lists, expensive computed fields and non-indexed search filters slow the UI. Design BCs for efficient queries and apply pagination where needed.

Example

A list with 1000+ records should use server-side pagination and efficient SearchSpecs to avoid long wait times.

Est. Time: 10 mins

Delivering UI Changes Safely

Use Workspaces and small incremental deliveries. Validate in Integration workspace before pushing to MAIN.

Checklist

  • Test in Dev workspace
  • Deliver to Integration for QA
  • Validate in a staging environment

Reasoning

Validation & Testing Strategy

Test UI changes with sample data, ensure BC mapping is correct and check for null/edge cases.

Test plan

  • Smoke test in Dev workspace
  • Functional test in Integration
  • Regression test in MAIN (if available)
Est. Time: 10 mins

Common Data Modeling Mistakes

Avoid making BCs dependent on heavy computed fields or joining too many tables; keep BCs focused and index-aware.

Troubleshooting

If a list is slow: check SearchSpec, check SQL generated by BC and add indexes where appropriate.
