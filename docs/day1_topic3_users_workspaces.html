User Accounts — Basic Concepts

Users are created in Administration. Each user has login credentials and assigned Responsibilities that determine the screens they can access.

What to set

Username, Email, Responsibility, Access Group and default Home page.

Est. Time: 12 mins

Responsibilities — Role Definitions

Responsibilities bundle views and screens into a role. They can be copied and adapted for different teams.

Example

Create a 'Service Agent' responsibility with access to case and queue views only.

Est. Time: 12 mins

Access Groups & Server Routing

Access Groups map users to server processes and determine available applets and performance characteristics.

Note

Set Access Group to align with server topology; use separate Access Groups for heavy workloads.

Est. Time: 10 mins

Workspace Types & Lifecycle

MAIN, Integration and Dev workspaces provide controlled staging for configuration changes.

Flow

Dev workspace: make changes → Integration: QA → MAIN: production. Use small, atomic changes to reduce conflicts.

Est. Time: 12 mins

Web Tools — Entry Points

Open Web Tools to manage BCs, BOs, Views, Applets and Workflows. Use Workspace selector to switch context.

Est. Time: 12 mins

Workflow Designer — Basics

Create workflows with Start/End nodes, Decisions, Business Service calls and Siebel Operations.

Example

Create a workflow that assigns high-priority cases to a dedicated queue and sends a notification.

Est. Time: 12 mins

Business Services — Reusable Logic

Business Services encapsulate server-side logic and can be invoked from workflows, client scripts or responsibilities.

Best practice

Keep Business Services generic and reusable; avoid embedding too much vertical logic in one service.

Est. Time: 10 mins

Assignment Manager & Queues

Assignment Manager routes records to queues based on rules and is essential for case distribution.

Example Rule

Route cases by territory to matching queues and escalate by response SLAs.

Est. Time: 10 mins

Security & Data Access

Combine Responsibilities, Field-level security, and record-level filters to protect sensitive data.

Tip

Test a user's experience by assigning the responsibility and using an audit/test account to verify permissions.

Est. Time: 10 mins

Workspace Delivery — Merge & Conflict Handling

When delivering to Integration, conflicts may appear if MAIN changed — resolve by merging carefully and testing.

Practical note

Always communicate delivery windows with other teams to avoid simultaneous changes.

Est. Time: 12 mins

Web Tools — Object Explorer Walkthrough

Object Explorer is where you locate BCs, BOs, Applets and Views for edits.

Change Management & Backouts

Keep small delivery bundles and take snapshots/backups before major changes to allow quick rollback.

Rollback plan

Document steps to revert changes and create a contact point for immediate support during delivery windows.

Est. Time: 8 mins

Developer Productivity Tips

Use naming conventions, comment fields and keep change logs in your workspace description for reviewers.

Example

Prefix custom fields with X_ and include your initials in workspace names for traceability.
