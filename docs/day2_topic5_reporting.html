üî® Creating Custom Reports

Report Builder Interface

Siebel's Report Builder provides a wizard-driven interface for creating custom reports without coding.

Report Creation Steps

  1. Select Data Source: Choose Business Component (e.g., Account, Opportunity)
  2. Choose Fields: Select columns to include in report
  3. Add Calculations: Sum, Count, Average, Min, Max
  4. Set Grouping: Group by owner, region, product, etc.
  5. Define Sorting: Order results by field(s)
  6. Apply Filters: Limit records (e.g., Status = Active)
  7. Format Layout: Headers, footers, fonts, colors
  8. Save and Test: Name report and run sample

Calculations: SUM(Revenue), COUNT(Opportunities)

üìâ Data Visualization Best Practices

Choosing the Right Chart Type

Bar/Column Charts: Compare values across categories

Example: Revenue by product line, SRs by priority

Line Charts: Show trends over time

Example: Quarterly revenue growth, daily SR volume

Pie Charts: Show proportional breakdown (use sparingly)

Example: Market share by competitor, SRs by type

Funnel Charts: Visualize conversion process

Example: Sales pipeline stages, lead conversion funnel

Gauge Charts: Show progress toward goal

Example: Quota attainment, SLA compliance %

Visualization Guidelines

  • Keep It Simple: Avoid cluttered charts with too many data points
  • Use Color Purposefully: Green = good, Red = bad, consistent meanings
  • Label Clearly: Axis labels, legends, data point values
  • Choose Appropriate Scale: Start Y-axis at zero for fair comparison
  • Limit Pie Slices: Maximum 5-7 slices, combine others into "Other"

‚ö° Report Performance Optimization

Slow Report Issues

Complex reports with large datasets can take minutes to run. Optimization is critical for user satisfaction.

Performance Bottlenecks

  • Large Data Volume: Millions of records
  • Complex Joins: Multiple related tables
  • Missing Indexes: Slow search fields
  • Calculations: Heavy processing in report
  • Concurrent Users: Many reports running simultaneously

Optimization Techniques

  1. Add Indexes: Index frequently queried fields
  2. Limit Date Range: Default to last 90 days instead of all time
  3. Use Parameters: Let users filter to relevant subset
  4. Pre-Aggregate Data: Create summary tables for dashboards
  5. Schedule Heavy Reports: Run overnight instead of on-demand
  6. Cache Results: Reuse recent report output
  7. Pagination: Display 100 rows at a time, not 10,000

Performance Targets

Excellent: < 3 seconds
Acceptable: 3-10 seconds
Slow: 10-30 seconds (needs optimization)
Very Slow: > 30 seconds (requires immediate attention)

üìã Report Governance

Managing Report Sprawl

Without governance, organizations end up with hundreds of redundant, outdated reports that confuse users.

Best Practices

  • Naming Standards: Clear, consistent names (e.g., "SLS_Pipeline_ByRep_Monthly")

  • Ownership: Assign owner responsible for maintenance

üîß Troubleshooting Reports

Common Report Issues

Problem: Report shows no data

Solutions:

  • Check user has access to underlying records
  • Verify filter criteria isn't too restrictive
  • Confirm data exists for selected date range
  • Review visibility rules and team assignments

Problem: Numbers don't match expectations

Solutions:

  • Verify report definition (correct fields, calculations)
  • Check for timezone issues with date/time fields
  • Confirm currency conversion if multi-currency
  • Review filter logic (AND vs OR conditions)
  • Ensure latest data has synced (refresh analytics DB)

Problem: Report times out or errors

Solutions:

  • Add date filter to limit records
  • Optimize query (add indexes, simplify joins)
  • Increase timeout setting
  • Schedule during off-hours instead of real-time
  • Break into smaller sub-reports

‚úÖ Reporting Best Practices Summary

Design Principles

  1. Know Your Audience: Executive summary vs analyst detail
  2. Start with Questions: What decisions will this report support?
  3. Keep It Simple: One primary message per report
  4. Use Visuals: Charts communicate faster than tables
  5. Provide Context: Include comparisons, targets, trends
  6. Enable Action: Reports should drive decisions
  7. Test Thoroughly: Validate accuracy before distribution

Performance Guidelines

  • Target report runtime: < 5 seconds for interactive, < 30 seconds for batch
  • Limit row count: 1,000 rows for screen, 10,000 for export
  • Index search fields used in filters
  • Cache frequently accessed data
  • Use analytics database for heavy reporting

Security Reminders

  • Apply row-level security (users see only their data)
  • Mask sensitive fields (SSN, credit cards)
  • Audit report execution
  • Encrypt emailed reports
  • Set appropriate export limits
