Siebel uses a classic 3-tier architecture: Presentation, Application, Data.

Three Tiers

1. Presentation Tier (UI)

  • Siebel Web Client: Runs in browser (HTML/JavaScript)
  • Siebel Mobile: Disconnected client for field users
  • Open UI: Modern JavaScript framework for customization

2. Application Tier (Business Logic)

  • Siebel Server: Processes requests, workflows, and rules
  • Object Manager: Manages business objects (BOs) and components (BCs)
  • Workflow Engine: Executes multi-step processes
  • Assignment Manager: Routes tasks to users/queues

3. Data Tier (Database)

  • Oracle DB, MS SQL, or DB2: Stores all customer and config data
  • Schema: Tables (S_ORG_EXT, S_CONTACT, S_SRV_REQ, etc.)
  • Indexes & Views: Optimize query performance

Why 3-Tier?

  • Scalability: Add more app servers without changing UI or DB
  • Maintainability: Update logic without touching presentation
  • Security: Isolate database from direct user access
